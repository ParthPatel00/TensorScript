Bottom line: For the specific workload TensorScript targets — CPU inference at batch size 1 with matmul + activation layers — it is faster than every alternative measured and uses 3× less peak memory than NumPy. It achieves this by eliminating the two main sources of framework overhead: per-operation Python dispatch latency and intermediate tensor allocation.
Where TensorScript loses: large-batch GEMM (BLAS already wins), expensive transcendentals like sigmoid/tanh on large arrays (PyTorch's MKL exp() is faster), and anything outside its supported op set (convolutions, attention, etc.).
